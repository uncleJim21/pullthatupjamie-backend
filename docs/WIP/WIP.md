## Recently Implemented (pending frontend integration)

- [x] **Synthesis-failure recovery tier system (DSML/narration/empty-text catch).** 2026-04-28. Three-tier reactive recovery wired in `routes/agentChatRoutes.js` after the primary synthesis pass. Quality gate at `utils/agent/synthesisQuality.js` evaluates `evaluateSynthesisOutput({ text, outputTokens })` against five triggers: `empty`, `low_tokens` (< 250 output tokens, env-overridable), `markup_residue` (post-sanitize DSML detection), `narration` (short text + leading "let me / I'll / let's grab" preamble), `truncated_clip` (`{{clip:` ending mid-token). Recovery cascade: **Tier 1** strict re-synthesis on same provider with `buildStrictSynthesisPrompt(intent)`, `temperature: 0`, `tool_choice: 'none'`, half budget; **Tier 2** cross-provider re-synthesis on Anthropic Haiku 4.5 (uses `AGENT_MODELS.fast`) — messages run through new `sanitizeMessagesForAnthropic()` to strip non-Anthropic-native blocks (DeepSeek's `thinking` blocks 400ed Anthropic with `messages.N.content.M.thinking.thinking: Field required`); **Tier 3** hardcoded `TIER3_FALLBACK_MESSAGE`. Aborted-but-clean primary handling: if primary errors mid-stream (synthesis-budget timeout) but the streamed text passes content checks, accept it instead of triggering redundant recovery. Recovery passes are silent (no `text_delta` emit) so the canonical `text_done` payload carries the recovered prose. Observability: per-tier records in `agentLog.synthesisRecovery`, `synthesis_tier{1,2,3}` rounds in `agentLog.rounds`, SSE `status` events ("Refining your answer…", "Trying a different model…"). Provider-side: `temperature` parameter added to `deepSeekProvider` and `anthropicProvider`. 20-query regression 2026-04-28: 0 DSML/markup leaks reaching user (was 3 prior); 2 Tier 2 recoveries fired correctly on proper-noun queries with thin evidence (lncurl.lol / ellen curl, primary emitted DSML-only with empty text content); Tier 3 never fired. Plan: [`SYNTHESIS_FAILURE_RECOVERY_PLAN.md`](./SYNTHESIS_FAILURE_RECOVERY_PLAN.md). Regression artifact: `tests/output/comparison-2026-04-28T00-*.md`.
- [x] **Synthesis `tool_choice: 'none'` (DSML root-cause fix).** 2026-04-27. The synthesis fallback was passing `tools: []`, which left DeepSeek in a contradictory state (history shows tool calls; current request says no tools exist) and caused it to inline its native `<｜DSML｜tool_calls｜...>` markup as plaintext. Per [DeepSeek's API guide](https://api-docs.deepseek.com/guides/function_calling), the documented way to forbid tool invocation while keeping the model anchored is `tool_choice: 'none'` *with* the tool schemas still attached. All four provider adapters (`deepSeekProvider`, `tinfoilProvider`, `openRouterProvider`, `anthropicProvider`) now accept a `toolChoice` parameter; OpenAI-compatible providers translate it directly, Anthropic translates `'none'` → `{ type: 'none' }`. Synthesis call site in `routes/agentChatRoutes.js` switched from `tools: []` to `tools: effectiveTools, toolChoice: 'none'`. Full rationale + history captured in [`docs/AGENT_SYNTHESIS_PASS.md`](../AGENT_SYNTHESIS_PASS.md). Companion runtime defenses (synthesis prompt guard + `sanitizeAgentText` streaming filter) unchanged — defense in depth.
- [x] **Synthesis token budget + modular response-length prompt.** 2026-04-27. Synthesis pass `maxTokens` raised 2048 → 4096 (the older 2048 cap was truncating long answers in the synthesis fallback path on DeepSeek; mid-token cut-offs observed in the wild). Now env-overridable via `AGENT_SYNTHESIS_MAX_TOKENS`. Response-format prompt split into `PROMPT_SECTIONS.responseFormatBase` (stable formatting/clip rules) plus swappable `responseLengthDefault` / `responseLengthExpansive` variants; selected at module load via `AGENT_RESPONSE_VERBOSITY` (`default` | `expansive`). Lays groundwork for per-request "go crazy" mode without re-templating the formatting rules. Clip-citation rule (rule 6) softened from "MANDATORY 3-5" to "at least 1 grounded in evidence, max 5, feel free to reference"; synthesis guard's matching "3-5 clips minimum suspended" carve-out reworded to "minimum-clip floor is suspended".
